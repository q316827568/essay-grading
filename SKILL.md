# 小学作文评分技能 v3.2

按标准评分细则对小学作文进行专业评分，支持多模态识图和并行评分优化。

## 核心更新（v3.2）

1. **仲裁模型升级**：GLM-5 → **DeepSeek-V3.2**（速度快3倍，约40秒 vs 140秒）
2. **去掉卷面分**：满分从 32 分调整为 **30 分**
3. **实时进度卡片**：每个模型完成后立即更新飞书卡片，无需等待全部完成

## 评分策略

### 普通模式
```
KIMI-2.5 多模态识图 → 直接输出评分结果
```

### 专家模式（并行优化 + DeepSeek仲裁）
```
┌─────────────────────────────────────┐
│  并行执行（~30-60秒）               │
│  ┌─────────────┐ ┌─────────────┐   │
│  │ KIMI-2.5    │ │ 豆包 Vision  │   │
│  │ 识图评分    │ │ 识图评分    │   │
│  └─────────────┘ └─────────────┘   │
└─────────────────────────────────────┘
              ↓
    DeepSeek-V3.2 仲裁（~40秒）
              ↓
        最终评分结果
              ↓
     飞书卡片消息（实时更新进度）
```

**进度卡片更新时机：**
1. KIMI 完成 → 卡片更新为"KIMI完成，豆包进行中..."
2. 豆包 完成 → 卡片更新为"豆包完成，DeepSeek仲裁中..."
3. 仲裁 完成 → 卡片更新为绿色"评分完成"

## 评分标准（满分30分，无卷面分）

| 维度 | 分值 | 评分要点 |
|------|------|----------|
| 内容质量 | 10分 | 切合题意、中心明确、内容充实、情感真实 |
| 语言表达 | 8分 | 语句通顺、用词准确、表达生动、无语病 |
| 结构层次 | 6分 | 条理清晰、段落分明、过渡自然、首尾呼应 |
| 书写规范 | 4分 | 字迹工整、标点正确、格式规范、错别字少 |
| 字数要求 | 2分 | 达到字数要求（400字以上） |

**等级划分：**
- 27-30分: 一类文·优秀
- 23-26分: 二类文·良好
- 17-22分: 三类文·中等
- 11-16分: 四类文·及格
- 0-10分: 五类文·不及格

## 模型配置

| 模型 | 用途 | 特点 |
|------|------|------|
| `Kimi-K2.5` | 多模态识图评分 | 中文识别准确，评分专业 |
| `doubao-1-5-vision-pro-32k-250115` | 多模态识图评分 | 豆包视觉模型 |
| `DeepSeek-V3.2` | 专家模式仲裁 | 速度快（40秒），判断准确 |

详细配置见 `references/models.md`

## 输出格式

1. 字数统计
2. 符合题意检查
3. 分项评分
4. 详细点评（优点 + 不足）
5. 总分及总结
6. （专家模式）不同专家的打分及差异及评审意见

## 飞书卡片消息

专家模式评分完成后自动发送飞书卡片消息，特点：

1. **卡片格式**：使用 `msg_type: "interactive"` 卡片消息
2. **颜色区分等级**：
   - 一类文 → 绿色 (green)
   - 二类文 → 蓝色 (blue)
   - 三类文 → 橙色 (orange)
   - 四类文/五类文 → 红色 (red)
3. **格式优化**：使用 lark_md 格式，加粗关键词，支持 emoji

## 重要注意事项

1. **飞书输出**：⚠️ terminal 输出不会发送到飞书，必须直接在回复中输出内容
2. **评分核心**：必须严格依据题目要求，核心要求未满足要重扣分
3. **题目示例**：题目示例的人物都符合题意，不能主观判定偏题
4. **JSON解析**：GLM-5 可能返回被 \`\`\`json...\`\`\` 包裹的JSON，需要先去除代码块标记
5. **并行实现**：使用 `concurrent.futures.ThreadPoolExecutor` 并行调用多模态模型
6. **表格格式**：飞书纯文本不支持表格，使用逐行格式如 `内容质量：KIMI 7分 | 豆包 6分 | 最终 7分`
